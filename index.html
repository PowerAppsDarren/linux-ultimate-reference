Persistent Environment Variables

To make environment variables permanent, add them to one of these files:

# For current user only
~/.bashrc              # Bash shell
~/.profile             # General profile
~/.bash_profile        # Bash login shell

# For all users (system-wide)
/etc/environment       # System environment
/etc/profile           # System profile
/etc/bash.bashrc       # System bash config

Example: Adding to PATH in ~/.bashrc:

echo 'export PATH=$PATH:/new/path' >> ~/.bashrc
source ~/.bashrc  # Reload the configuration

SSH Configuration

Create/edit ~/.ssh/config for easier connections:

Host myserver
    HostName server.example.com
    User john
    Port 2222
    IdentityFile ~/.ssh/id_rsa_server

Host github
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_github

Then connect simply with:

ssh myserver
ssh github

Useful commands for log analysis:

# Count occurrences of IP addresses in access log
awk '{print $1}' /var/log/apache2/access.log | sort | uniq -c | sort -rn

# Find most common 404 errors
grep " 404 " /var/log/apache2/access.log | awk '{print $7}' | sort | uniq -c | sort -rn

Monitor system resources with watch:

watch -n 2 free -m
watch -n 2 df -h
watch -n 1 'ps aux | sort -nrk 3,3 | head -n 5'  # Top 5 CPU users

Create a simple monitoring script:

#!/bin/bash
echo "=== System Performance Report ==="
echo "Date: $(date)"
echo ""
echo "CPU Load:"
uptime
echo ""
echo "Memory Usage:"
free -h
echo ""
echo "Disk Usage:"
df -h
echo ""
echo "Top 5 Processes:"
ps aux | sort -nrk 3,3 | head -n 6

Advanced Tips and Tricks

Command Chaining

Operator Description Example
cmd1 ; cmd2 Run cmd1 then cmd2 echo "Hello" ; echo "World"
cmd1 && cmd2 Run cmd2 only if cmd1 succeeds mkdir dir && cd dir
cmd1 || cmd2 Run cmd2 only if cmd1 fails ping -c1 google.com || echo "Offline"
cmd1 | cmd2 Pipe output of cmd1 to cmd2 ls -l | grep ".txt"
cmd & Run cmd in background firefox &
(cmd1 ; cmd2) Group commands (cd dir && ls -l) | grep ".txt"

Redirections

Operator Description Example

Cron Jobs

Edit crontab with:

crontab -e

Cron job format:

minute hour day_of_month month day_of_week command

Examples:

# Run daily at 3:30 AM
30 3 * * * /path/to/script.sh

# Run every Monday at 9:00 AM
0 9 * * 1 /path/to/script.sh

# Run every 15 minutes
*/15 * * * * /path/to/script.sh
